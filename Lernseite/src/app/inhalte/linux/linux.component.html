Linux

  • Befehle

    • ls : Inhalt eines Verzeichnisses auflisten
      • -l : detailliert
      • -a/--all: auch versteckte
      • -R : rekursive Ausgabe aller Unterverzeichnisse
      • -h/--help : Hilfsinformationen
      • --color : farbige Ausgabe
      • Spezifikationen können kombiniert werden
    • cp - kopiert Dateien/Verzeichnisse
    • mv - verschiebt Dateien/Verzeichnisse
    • rm - löscht Dateien/Verzeichnisse
    • touch - erzeugt leere Datei/aktualisisert Zeitstempel, wenn Dateiname schon existiert
    • nano/vi/emacs - Editoren zur Bearbeitung von Dateien
    • less - Ausgabe von Dateien auf Bildschirm
    • pwd - gibt Namen des aktuellen Arbeitsverzeichnisses
    • cd - Wechsel des Arbeitsverzeichnisses
    • mkdir - Verzeichnis erzeugen
    • rmdir - Verzeichnis löschen
    • ps - Prozesse auflisten
      • -a/all - alle Prozesse (die an Terminal gebunden sind)
      • -l/long - Langformat
      • -f vollständige Ausgabe
      • -F - noch vollstängigere Ausgabe
      • -x - Auch Prozesse, die nicht an Terminal gebunden sind
    • man Befehl- Informationen zu Befehl
    • file : Typ der Datei
    • kill/Ctrl+C : Abbruch eines Prozesses
    • who : angemeldeter Benutzer
    • alias : Name für wiederkehrenden Befehl zuordnen
    • echo : Ausgabe der Argumente
    • wc : word Count/Anzahl Wörter
    • head : AUsgabe erste Zeile
    • History:
      • history : Ale Befehle mit Nummer
      • !"Nummer" : Befehl mit der "Nummer" nochmal
      • !"text" : Letzter Befehl, der mit "text" beginnt nochmal
    • Wildcards:
      • * : Beliebige Zeichenkette
      • ? : Beliebiges einzelnes Zeichen
      • [0-9] : Beliebiges Zeichen Zwischen 0 und 9
    • Quotierung:
      • \ : Maskieren des nächsten Zeichens
      • '...' : Maskieren aller eingeschlossenen Zeichen
    • Umlenkung Ein-/Ausgabe
      • befehl >(Andersum) Datei : Umleitung Standardeingabe
      • Befehl > Datei : Umleitung Standardausgbae, Überschreiben
      • Befehl >> Datei : Umleitung Standardausgabe, Anhängen
      • Befehl 2> Datei : Umleitung Standardfehlerausgabe
      • Verkettung: Pipes | : Standardausgabe des ersten Kommandos auf Standardeingabe des zweitenb Befehls

  • Dateiattribute

    • Berechtigungen
      • Zugriffsrechte für Benutzerklassen unterschiedlich
        • User - Besitzer
        • Group - Besitzende Gruppe
        • Others - Restliche Welt
      • Verschiedene Rechte:
        • r : Read (Lesen/ Verzeichnis kann angezeigt werden)
        • w : Write (Schreiben/ Neue Dateien/ Löschen)
        • x : Execute (Ausführen/ Öffnen bei Ordnern)
      • Schreibweisen/ Zuweisung
        • Dreistellige Oktalzahl
          • Werte für Benutzerklassen als Binärzahlen nach rwx (Bsp. rx = 101)
          • Werte für Benutzerklassen als Oktalzahlzahlen (Bsp. 101 = 4+0+1 = 5)
          • Werte für alle Benutzerklassen hintereinanderschreiben: user, group, others (Bsp. U:5,G:1,O:0 = 510
        • Symbolische Angabe
          • who operator permission
            • who: u(User), g(Group), o(Others), a(All)
            • operator: +(hinzufügen), -(entfernen), =(gleichsetzen)
            • permission: r(read), w(write), x(execute)
      • SUID-Bit(Set User Id Bit)
        • Bei Programmen
        • Programm wird mit Rechten des Besitzers ausgeführt (Nicht wie Ausführer)
        • Oktalwert: 4000, symbolisch: s bei User
      • SGID-Bit (Set Group Id Bit)
        • Bei Programmen: Wird mit Rechten der Gruppe ausgeführt
        • Bei Verzeichnissen: Wird mit Gruppe des Verzeichnisses erstellt
        • Oktalwert: 2000, symbolisch: s bei group
      • Sticky Bit
        • Verzeichnis: nur Eigentümer einer Datei darf Datei löschen
        • Oktalwert: 1000, symbolisch: t
      • Befehle:
        • chmod : Ändern der Zugriffsrechte
        • chown : Ändern des Besitzers
        • chgroup : Ändern der Gruppe
        • umask : Maske mit Zugriffsrechten setzen
          1. Rechte, wie man sie haben will: zB. umask: 027 = 000 010 111
          2. bitweises NICHT aus Standard umask: 111 101 000 = 750
          3. Wird bitweise UND mit Default: zB. UND 666 => 110 100 000 = 640
            • Default für Dateien: 110 110 110 = 666
            • Default für Verzeichnisse: 111 111 111 = 777

  • Prozesskontrolle

    • Prozess im Vordergrund blockiert andere Prozesse
    • & anhängen an Aufruf : Prozess im Hintergrund starten
    • pstree : Prozessbaum anzeigen
    • ctrl-Z : Prozess anhalten
    • bg/fg : Prozess in Hintergrund/Vordergrund bringen
    • jobs : Ausgabe der Job von Shell gestartet

  • Umgebungsvariablen

    • Bearbeiten
      • VARIABLE=INHALT : Setzen der Umgebungsvariable
      • echo $VARIABLE : Ausgabe von VARIABLE
      • printenv : Anzeigen der Umgebungsvariablen
      • unset VARIABLE : Löschen von VARIABLE
      • export VARIABLE : Bekannt machen für Kindprozesse der Shell
    • PATH : Befehlssuchpfad
    • HOME : Home-Verzeichnis
    • TERM : Terminal-Definition
    • PS1 : Login Prompt
    • PWD : Aktuelles Verzeichnis
    • OLDPWD : Vorheriges aktuelles Verzeichnis
