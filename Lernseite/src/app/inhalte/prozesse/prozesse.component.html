  • Prozesswechsel
    • Dispatcher
    • Scheduler
      • nicht päembtiv(nicht verdränged)
        • Prozess behält Kontrolle, bis fertig
        • Problem: Blockieren des Prozesses, wichtige Prozesse kommen nicht dran
      • präembtiv (verdränged)
        • CPU kann vor Fertigstellung entzogen werden
        • pasuieren, bis wieder CPU zugeteilt
        • Problem: Mehr Kontrolle nötig(Overhead)
        • Vorteil: Prioritäten beachtet
      • Verfahren
        • First Comes First Serve
        • Shortest Process next
        • Round-Robin
        • Prioritäten
        • Multi-Level Scheduling
        • Multi-Level Feedback Scheduling
        • Linux Scheduling
  • Prozesslebenszyklus
    • Created -Start dann Ready
    • Ready - zur Ausführung gewählt dann Running
    • Running - Zeit abgelaufen, dann Ready, oder Systemaufruf, dann bocked oder beendet, dann terminated
    • Blocked - Systemaufruf beednet, dann ready
    • terminated - Ende
  • Przesserzeugung
    • Windows
      • Prozessvergabelung - fork
        • Prozess wird dupliziert
        • erzeugender Prozess bleibt erhalten
    • Unix
      • Prozessverkettung - exec
        • ein Prozess erzeugt einen neuen
        • erzeugender Prozess beendet sich
      • Prozesserzeugung
        • Prozess dupliziert sich (fork)
        • neuer Prozess ersetzt sich selber mit exec
        • Alter Prozess bleibt erhalten und komplett neuer Prozess existiert auch
