Von Neumann

  • Prinzipien

    1. 4 Werke
      1. Rechenwerk - arithemtishe/logische Funktionen
      2. Leitwerk - Interpretiert Programme
      3. Speicher - Hauptspeicher/Arbeitsspeicher für Programme/Daten
      4. Ein-/Ausgabewerk - Kommuniziert mit Umwelt, Langzeitspeicher
    2. Programmgesteuert - Daten + Befehle(von Befehlsinterpreter interpretiert) -> Ergebnisse
    3. Programme und Daten im selben Speicher, modifizierbar
    4. Hauptspeicher mit gleich großen Adresszellen
    5. Programm = Folge von Befehlen (Sequentiell)
    6. Abweichungen von Sequenzieller Ausführung durch:
      • Sprung an bestimmte Zelle
      • von Auswertung der gespeicherten Werte abhängig
    7. Duale Darstellung von Zahlen

  • Von Neumann heute

    • weitestgehend unverändert
    • Komponenten mehrfach für Parallelverarbeitung (zB Rechenwerk)
    • Sequenzielle Ausführung durch Nebenläufigkeit ergänzt
    • viertsufige Speicherhierarchie: Haupt- und Sekundärspeicher
    • Nur auf unterster Ebene verschiedene Speicher für Daten und Programme -> gleichzeitiger Zugriff über verschiedene Busse

  • Von-Neumann Architektur

    • Von-Neumann-Zyklus
      • FETCH: Befehl aus Speicher in Befehlsregister
      • DECODE: Steuerwerk wandelt Befehl in Schaltinstruktion für Rechenwerk um
      • FETCH OPERANDS: Evtl Parameter aus Speicher holen
      • EXECUTE: Ausführung(Rechenwerk)
      • UPDATE PROGRAM COUNTER: Befehlszähler auf nächsten Befehl
      • WRITE BACK: Ergebnis in Hauptspeicher/an Ausgabegerät
    • Bussysteme
      • Datenbus
        • In beide Richtungen
        • Überträgt Daten zwischen CPU, Arbeitsspeicher und Peripherie
        • Anzahl der Datenbusleitungen legt Übertragungsgeschiwindigkeit der Bytes pro Takt fest - Meist = größe des Arbeitsregister im Rechenwerk (meist 64 Bits)
      • Adressbus
        • Übertragt Speicheradressen
        • Speicheradressen und Peripherie-Geräte werden angesprochen
        • Anzahl Busleitungen = maximale Anzahl Speicheradressen - 32 Leitungen -> 32 Bit Lange Adressen -> 4GB Speicher
      • Steuerbus
        • Überträgt Kommandos von CPU an Peripherie-Geräte
        • Komponenten werden von Adressbus agesprochen und über Steuerbus angewiesen
        • Enthält auch Interrupt-Leitungen für I-/O-Geräte
        • Bandbreite: Meist 10 Leitungen
      • Moderne Entwicklung
        • Geräte nicht mehr direkt mit CPU verbunden
        • Chipsatz wird in CPU verlagert
        • verschiedene serielle und parallele Bussysteme
        • Controller als Vermittler zwischen Gerät und CPU
        • Northbridge
          • Dicht an CPU
          • Schnelle Datenübertragung
          • Anbindung Hauptspeicher und Grafikkarte an CPU
        • Southbridge
          • langsamere Verbindungen
        • Front-Side-Bus (FSB)
          • Bus zwischen CPU und Chipsatz
          • Enthält Adressbus, Datenbus und Steuserbus
    • Ein-/Ausgabe
      • Einlesen von Daten:
        • Busy Waiting
          • Treiber sendet ANfrage und wartet in Endlos-Schleife, bis Daten da sind
          • + Keine zusätzliche Hardware
          • - Belastet CPU
        • Interrupt gesteuert
          • Man braucht: Interrupt-COntroller und Leitungen im Steuerbus für Interrupt
          • Treiber sendet Aufgabe und schläft
          • Interrupt weckt Treiber wieder auf
          • + CPU ist nicht blockiert
          • - Zusätzliche Hardware nötig
        • Direct Memory Access
          • Man braucht: DMA-Controller
          • Daten direkt zwischen Arbeitsspeicher und I-/O-Gerät übertragen
          • zB HDD/SSD, Soundkarte, Netzwerkkarte
          • Nach Übertragung wird Interrupt ausgelöst
          • + Entlastung der
          • - Zusätzliche Hardware(DMA-Controller) - In Chipsatz integriert
      • Zeichenbasiert - Einzelne Zeichen direkt kommunniziert(zB Maus, Tastatur, Drucker)
      • Blockorientiert - Datenübertragung erst nach Block (zB Festplatte, SSD, Laufwerke), Unterstützen meist Direct Memory Access (DMA) - Daten ohne CPU übertragen
      • Daten lesen:
        1. Prozess fordert von CPU Daten von Festplatte
        2. CPU schickt mit Treiber einen I-/O-Befehl an Controller
        3. Controller suchte Daten
        4. Prozess erhält Daten
    • Speicher
      • Pyramide
        • hohe Kosten
        • schneller Zugriff
        • geringe Kapazität
        1. I Register
        2. I interner Cache
        3. I externer Cache
        4. I Hauptspeicher(RAM)
        5. II SSD/ CompactFlash
        6. II Festplatten
        7. III Optische Laufwerke
        8. III Magnetoptische Laufwerke
        9. III Magnetbänder
        • Write-Trough:
          • Sofort an tiefere Schichten weitergegeben
          • + Konsistenz
          • - Geschwindigkeit
        • Write-Back:
          • Erst weitergegeben, wenn aus Cache verdrängt
          • + Geschwindigkeit
          • - Änderungen bei Systemausfall verloren
      • Register
        • Daten, auf die CPU direkt zugreifen kann
        • Genauso schnell wie CPU
        • Datenregister - Operanden für ALU und deren Resultate
        • Adressregister - Speicheradressen von Operanden und Befehlen
        • Befehlszähler - Speicheradresse des nächsten Befehls
        • Befehlsreigster - sktueller Befehl
        • Stapelregister - Speicheradresse am Ende des Stacks
      • Caching
        • Kopien von Teilen des Arbeitsspeichers
        • Beschleunigung von Datenzugriff
        • Verschiedene Level
          • First Level Chache - In CPU integriert (4-256kB)
          • Second Level Cache - langsamer und größer (256kB-4MB)
          • Third Level Cache - Außerhalb CPU (1MB-16MB)
