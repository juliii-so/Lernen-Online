Codierung

Inhalte

  • Anwendung von Umrechnungen
  • Verstehen von Negationen
    • Translationsdarstellung:
      • Es gibt keine negativen Zahlen
      • Stattdessen wird X+T dargestellt - Traslation/Bias wird hinzugefügt
      • T=1/2*B oder T=1/2*B-1 (Je nachdem, ob 0 positiv oder negativ sein Soll)
      • Ergibt Zahlenraum: -2^n-1 ... 2^n-1 -1 oder -(2^n-1 -1) ... 2^n-1
      • Mit 4 Bit kann
    • Komplementdarstellung
      • Einerkomplement
        • Negative Zahlen werden durch Invertierung dargstellt
      • Zweierkomplement
        • negative Zahlen werden durch Invertierung und +1 Dargstellt
        • Keine Doppelte 0 - Größere Zahlenraum
  • Reele Zahlen
    • Festkommadarstellung:
      • Vorkommateil: (ganz Normal) Zahl/Basis=Neue Zahl, Rest wird von unten nach oben gelesen
      • Nachkommateil: Zahl*Basis=Ergebnis, bis neue Zahl 0, Vorkommastelle von Ergebnis ist Binär von oben nach unten, 0,Nachkommastelle wird neue Zahl
      • Kann mit festem Skalierungsfaktor mit Kommaverschiebung als Ganzzahl gespeichert werden
      • Strichrechnung normal, Punktrechnung braucht Korrektur
    • Gleitkommazahlen(IEEE-Standard)
      • einfache Genauigkeit: float (32Bit)
      • doppelte Genauigkeit: double (64Bit)
      • Halblogarithmische Zahlendarstellung: X0 Xs * B^E
        • Xs= Mantisse(Signifikant)
        • B= Basis (meist 10)
        • E= Exponent(ganzzahlig)
      • Rechnerinterne Form: S-C-M
        • S = Vorzeichen
        • C = Charakteristik (Eponent in Translationsdarstellung: C=E+T - T=2^(r-1)-1
        • M = normalisierte Mantisse (ohne Vorzeichen)(Normalisiert: Erste Vorkommazhla ist erste ziffer ungleich 0 ) (optimiert: Vorkommzahl ist 1, kann dann weggelassen werden)
        • Basis ist 2
        • X0 (-1)^S*1,M*2^(C-T)
